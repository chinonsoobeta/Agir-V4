# Unsupervised Demo Script

Use this one-page script when a professional is testing Agir without live
builder guidance. The demo is for evaluation only; no investment decision should
be made solely from Agir output.

No investment decision may be recorded from a pilot workspace without human
review and firm approval.

## Before Starting

1. Open the sandbox URL and sign in with the demo credentials provided in the
   invite.
2. Create workspace only if the evaluator was not assigned a prepared sandbox
   workspace.
3. Start on Dashboard and read the "Unsupervised demo guide" card.
4. Open Settings -> Data & privacy and review the enterprise trust controls.
5. Confirm the tester understands this is decision-support, not a production
   investment approval system.

## Required Demo Path

1. Click "Seed demo deal" and choose Harbour Centre.
2. Open the seeded Harbour Centre deal.
3. Confirm the source documents are visible, including the construction budget,
   lender term sheet, and rent roll.
4. Review assumptions and verify the exit-cap conflict is visible instead of
   silently resolved.
5. Resolve conflicts using source text and conservative policy.
6. Accept documented defaults only when the source package is genuinely missing
   a required input.
7. Run underwriting and inspect the main metrics.
8. Use provenance/explanation affordances on at least five figures before
   trusting the output.
9. Generate IC memo and stakeholder reports.
10. Download memo/report artifacts. Export the customer audit package.
11. Optional real-deal path: Upload source documents for one nonconfidential
    deal and record extraction or modeling mismatches.

## Success Criteria

- Time-to-underwriting is less than 45 minutes for a prepared package.
- No support intervention is required for upload, extraction, assumption review, underwriting, memo, or audit export.
- User can explain why the top five metrics are trusted.
- User can identify which controls are implemented and which require external SOC 2, pen-test, SSO, SCIM, legal, or operations work.
- User can describe one workflow they would want before using Agir on a live
  investment committee process.

## Failure Criteria

- User cannot find the next action.
- User cannot distinguish approved, pending, defaulted, and conflicted assumptions.
- User cannot trace a displayed metric to source inputs.
- Extraction silently fabricates a required assumption.
- Audit package export is missing source documents, assumptions, reports, memo snapshots, or audit logs.

## Feedback To Capture

- What confused you?
- Which number or output looked least trustworthy?
- Which step would block use on a real, nonconfidential deal?
- Would you ask an analyst to review another deal in Agir?
